How AyuAware Intelligence Works

AyuAware turns wearable signals, user-fed inputs, and exposure context into traceable health readings, Project decisions, NSL actions, and timely nudges.

Inputs start in three lanes

This is the simple model for clients. Underneath, each lane still has to prove freshness and traceability before it can influence the app.

W

Wearable

Sleep, HRV, resting heart rate, recovery, activity, SpO2, and device sync trends.

U

User-fed

Onboarding, daily check-ins, tongue scans, meals, symptoms, labs, supplements, and NSL feedback.

E

Exposure

Weather, AQI, humidity, season, lunar, and astro/Panchang context acting on the Twin.

Pipeline at a glance

The main page stays intentionally light. Details move into the trace panel so the system feels readable before it feels technical.

When does this run?

The same backend spine runs at key moments instead of creating one-off shortcuts.

1
Onboarding

Builds the first baseline, current-state picture, Project, and plan.

2
Daily check-in

Refreshes interpretations, score context, triggers, and plan state.

3
Wearable sync

Periodically brings passive device data into the same evidence chain.

4
New scan/report

Routes tongue, lab, meal, supplement, and symptom inputs through processing.

5
Plan window

Maintains, modifies, or regenerates Project/NSL when rules require it.
